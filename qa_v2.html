QA Automation Java Interview Preparation Guide

1. Аннотації (Annotations)

Мітки метаданих, що додаються до коду для надання додаткової інформації компілятору чи середовищу виконання.


// Приклад анотацій
@Override // Перевизначення методу батьківського класу
@Test // Позначення тестового методу в JUnit
@Deprecated // Позначення застарілого методу
public void myMethod() {
    // Код методу
}

2. Rest Assured

Бібліотека для тестування REST API в Java


given()
    .baseUri("https://api.example.com")
    .when()
    .get("/users")
    .then()
    .statusCode(200)
    .body("size()", greaterThan(0));

3. Selenide

Фреймворк для автоматизації тестування на основі Selenium WebDriver


// Приклад використання Selenide
@Test
public void testLogin() {
    open("https://example.com");
    $("#username").setValue("testuser");
    $("#password").setValue("password");
    $("#login-button").click();
    $(".success-message").shouldBe(visible);
}

4. Паралелізація (Parallelization)

Виконання тестів одночасно для прискорення процесу тестування


// Приклад у JUnit 5
@Execution(ExecutionMode.CONCURRENT)
public class ParallelTests {
    @Test
    public void test1() { /* ... */ }

    @Test
    public void test2() { /* ... */ }
}

5. CI/CD (Continuous Integration/Continuous Deployment)

Практика автоматизації процесів збірки, тестування та розгортання додатків


# Приклад Jenkins pipeline
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                mvn clean package
            }
        }
        stage('Test') {
            steps {
                mvn test
            }
        }
        stage('Deploy') {
            steps {
                deploy to: 'production'
            }
        }
    }
}

7. Kubernetes (k8s)

Система управління контейнеризованими додатками


# Приклад deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: my-app:latest

10. Логи (Logs)

Запис подій та повідомлень для діагностики


// Використання Log4j
private static final Logger logger = LogManager.getLogger(MyClass.class);

public void myMethod() {
    logger.info("Початок методу");
    try {
        // Код методу
        logger.debug("Проміжний стан");
    } catch (Exception e) {
        logger.error("Помилка: ", e);
    }
}

11. Ретраї (Retries)

Повторні спроби виконання операції


// Приклад ретраю в тестуванні
@Test
@Retryable(maxAttempts = 3,
           backoff = @Backoff(delay = 1000))
public void testFluckyOperation() {
    // Код, який може fail'нутися
}

12. Техніки Тест-Дизайну

  • Еквівалентне розділення: Групування вхідних даних на класи
  • Аналіз граничних значень: Тестування на межах діапазонів
  • Таблиця рішень: Тестування комбінацій умов

19. ООП (Об'єктно-Орієнтоване Програмування)

Парадигма програмування, що базується на концепції "об'єктів"


public class Animal {
    private String name;

    public void makeSound() {
        System.out.println("Some sound");
    }
}

public class Dog extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Bark");
    }
}
